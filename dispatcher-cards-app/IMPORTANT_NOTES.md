Проблема: Next.js Hot Module Replacement (HMR) иногда откатывает изменения в файлах во время работы dev-сервера.

## ✅ Решение:

1. **Перед изменением файлов** - останови dev-сервер
2. **Внеси изменения** в файлы
3. **Запусти dev-сервер** заново
4. **Проверь результат** в браузере

## 🔧 Команды для работы:

```bash
# Остановить сервер
controlPwshProcess action="stop" terminalId="X"

# Запустить сервер
controlPwshProcess action="start" command="npm run dev" cwd="dispatcher-cards-app"

# Проверить вывод сервера
getProcessOutput terminalId="X" lines=30
```

## 📱 Проверка результата:

- URL: http://localhost:3000
- Полная перезагрузка: Ctrl+Shift+R (Windows) или Cmd+Shift+R (Mac)

## 🎯 Что НЕ делать:

❌ НЕ переписывай файлы без остановки сервера
❌ НЕ используй `strReplace` если файл откатился - останови сервер и перепиши
❌ НЕ добавляй подзаголовок обратно
❌ НЕ увеличивай размер заголовка
❌ НЕ уменьшай отступы

## ✅ Что делать при проблемах:

1. Проверь, что сервер запущен: `listProcesses`
2. Проверь вывод сервера: `getProcessOutput`
3. Если есть ошибки компиляции - исправь их
4. Если изменения не применились - останови и запусти сервер заново
5. Если файл откатился - останови сервер, перепиши файл, запусти сервер
