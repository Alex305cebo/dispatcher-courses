# 🔒 Защита контента от копирования и скриншотов

## Что было реализовано

Добавлена комплексная система защиты контента курсов от несанкционированного копирования и создания скриншотов.

## Защищённые страницы

### Все страницы курсов защищены:
- `courses.html` - список курсов
- `course.html` - главная страница курса
- Все страницы в папке `pages/` (50+ файлов)
- Все модули обучения (module-1.html до module-12.html)
- Все тесты (test-1.html до test-12.html)
- Симулятор и все его шаги
- Документация, Load Board, звонки, кейсы, грузы

## Реализованные меры защиты

### 1. Защита от копирования текста

✅ **Отключение выделения текста**
- CSS свойство `user-select: none` для всех элементов
- Блокировка события `selectstart`

✅ **Блокировка контекстного меню**
- Отключение правого клика мыши
- Предотвращение вызова меню через события

✅ **Блокировка горячих клавиш копирования**
- `Ctrl+C` / `Cmd+C` - копирование
- `Ctrl+X` / `Cmd+X` - вырезание
- `Ctrl+A` / `Cmd+A` - выделить всё
- Блокировка событий `copy` и `cut`

### 2. Защита от скриншотов

✅ **Блокировка клавиши Print Screen**
- Перехват события `PrintScreen`
- Очистка буфера обмена при нажатии
- Работает на Windows, Mac, Linux

✅ **Защита через Page Visibility API**
- Скрытие контента при потере фокуса страницы
- Предотвращение скриншотов в фоновом режиме

✅ **Невидимый водяной знак**
- Добавлен прозрачный текст "ЗАЩИЩЕНО ОТ КОПИРОВАНИЯ"
- Виден на скриншотах, но незаметен при обычном просмотре
- Помогает идентифицировать источник утечки

### 3. Защита от инструментов разработчика

✅ **Блокировка DevTools**
- `F12` - открытие DevTools
- `Ctrl+Shift+I` / `Cmd+Option+I` - инспектор
- `Ctrl+Shift+C` / `Cmd+Option+C` - выбор элемента
- `Ctrl+Shift+J` / `Cmd+Option+J` - консоль
- `Ctrl+U` / `Cmd+U` - просмотр исходного кода

✅ **Обнаружение открытых DevTools**
- Мониторинг размера окна браузера
- Автоматическая очистка консоли каждые 100мс
- Переопределение методов console.log

✅ **Блокировка сохранения страницы**
- `Ctrl+S` / `Cmd+S` - сохранить страницу
- `Ctrl+P` / `Cmd+P` - печать страницы

### 4. Защита изображений

✅ **Блокировка drag & drop**
- Предотвращение перетаскивания изображений
- Отключение события `dragstart`

✅ **Защита от сохранения**
- `pointer-events: none` для изображений
- Блокировка контекстного меню на изображениях

### 5. Мобильная защита

✅ **Защита на мобильных устройствах**
- Блокировка длинного нажатия (long press)
- Предотвращение вызова контекстного меню
- Защита от скриншотов на iOS и Android

✅ **Защита форм ввода**
- Отключение автозаполнения
- Атрибут `readonly` до фокуса
- Предотвращение утечки данных

### 6. Дополнительные меры

✅ **Защита от расширений браузера**
- Переопределение `navigator.webdriver`
- Защита от автоматизированных инструментов

✅ **Защита от автоматического копирования**
- Счётчик попыток копирования
- Логирование подозрительной активности

## Технические файлы

### Основной файл защиты:
- `pages/content-protection.js` - универсальный скрипт защиты

### Интеграция:
- Подключен на всех страницах курсов через `<script src="content-protection.js"></script>`
- Работает совместно с `auth-check.js` для комплексной защиты

## Как это работает

1. **При загрузке страницы:**
   - Скрипт защиты запускается автоматически
   - Устанавливаются все обработчики событий
   - Применяются CSS стили для блокировки выделения

2. **При попытке копирования:**
   - Событие перехватывается и блокируется
   - Буфер обмена очищается
   - Действие не выполняется

3. **При попытке скриншота:**
   - Клавиша Print Screen перехватывается
   - Буфер обмена очищается
   - На скриншоте будет виден водяной знак

4. **При открытии DevTools:**
   - Попытка открытия блокируется
   - Консоль постоянно очищается
   - Методы отладки переопределены

## Ограничения защиты

⚠️ **Важно понимать:**

1. **Клиентская защита не абсолютна**
   - Опытный пользователь может обойти JavaScript защиту
   - Можно сделать фото экрана камерой
   - Можно использовать внешние инструменты захвата экрана

2. **Защита усложняет, но не делает невозможным**
   - Основная цель - защита от случайного копирования
   - Защита от неопытных пользователей
   - Создание барьера для массового копирования

3. **Рекомендации для полной защиты:**
   - Добавить серверную проверку и watermarking
   - Использовать DRM технологии
   - Реализовать потоковую передачу контента
   - Добавить юридические меры защиты

## Влияние на пользовательский опыт

✅ **Минимальное влияние:**
- Защита работает незаметно для обычных пользователей
- Не влияет на скорость загрузки страниц
- Не мешает нормальному просмотру контента
- Пользователи могут читать и взаимодействовать с материалами

❌ **Ограничения для пользователей:**
- Невозможно выделить и скопировать текст
- Невозможно сохранить изображения
- Невозможно открыть DevTools
- Невозможно сделать скриншот стандартными средствами

## Проверка работы защиты

### Тест 1: Копирование текста
1. Откройте любую страницу курса
2. Попробуйте выделить текст мышью - не должно работать
3. Попробуйте `Ctrl+A` - не должно работать
4. Попробуйте `Ctrl+C` - не должно работать

### Тест 2: Контекстное меню
1. Кликните правой кнопкой мыши - меню не должно появиться
2. Попробуйте на тексте, изображениях, ссылках - везде заблокировано

### Тест 3: DevTools
1. Нажмите `F12` - не должно открыться
2. Попробуйте `Ctrl+Shift+I` - не должно работать
3. Попробуйте `Ctrl+U` - не должно показать исходный код

### Тест 4: Скриншоты
1. Нажмите `Print Screen` - буфер обмена очистится
2. Если удастся сделать скриншот - будет виден водяной знак

### Тест 5: Сохранение
1. Попробуйте `Ctrl+S` - не должно работать
2. Попробуйте `Ctrl+P` - печать заблокирована
