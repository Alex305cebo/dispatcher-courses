# 🔒 Защита доступа к курсам

## Что было сделано

Добавлена система защиты доступа к курсам для незарегистрированных пользователей.

## Защищённые страницы

### Основные страницы курсов:
- `courses.html` - список всех курсов
- `course.html` - главная страница курса

### Страницы в папке `pages/`:
- `documentation.html` - документация
- `modules.html` - список модулей
- `module-1.html` до `module-12.html` - все 12 модулей
- `simulator.html` - симулятор диспетчера
- `simulator-step1.html` до `simulator-step4.html` - шаги симулятора
- `simulator-instruction.html` - инструкции симулятора
- `testing.html` - тестирование
- `test-1.html` до `test-12.html` - все 12 тестов
- `calls.html` - тренировка звонков
- `cases.html` - практические кейсы
- `loads.html` - примеры грузов
- `loadboard.html` - Load Board тренажёр

## Как это работает

1. **На странице `courses.html`:**
   - Незарегистрированные пользователи видят курсы, но не могут на них кликнуть
   - Кнопки "Начать" заменяются на "🔒 Требуется вход"
   - При клике показывается сообщение о необходимости входа
   - Пользователь перенаправляется на страницу `login.html`

2. **На странице `course.html` и всех страницах в `pages/`:**
   - При попытке открыть страницу проверяется авторизация
   - Если пользователь не авторизован, показывается сообщение
   - Автоматическое перенаправление на `login.html`

3. **Для авторизованных пользователей:**
   - Полный доступ ко всем материалам курсов
   - Имя пользователя отображается в навигации
   - Доступ к личному кабинету

## Технические детали

### Файлы системы защиты:
- `auth.js` - основной скрипт авторизации (корневая папка)
- `pages/auth-check.js` - скрипт проверки для страниц курсов

### Функции:
- `checkAuth()` - проверка наличия токена авторизации
- `requireAuth()` - обязательная проверка с перенаправлением
- `updateAuthUI()` - обновление интерфейса для авторизованных пользователей
- `logout()` - выход из системы

## Что нужно для доступа к курсам

Пользователь должен:
1. Зарегистрироваться на странице `register.html`
2. Войти в систему на странице `login.html`
3. После входа получить доступ ко всем курсам

## Проверка работы

1. Откройте `courses.html` без авторизации - кнопки должны быть заблокированы
2. Попробуйте открыть любую страницу из `pages/` - должно перенаправить на `login.html`
3. Войдите в систему через `login.html`
4. Теперь все курсы должны быть доступны

## Безопасность

- Токен авторизации хранится в `localStorage`
- Проверка выполняется на клиентской стороне
- Для полной безопасности рекомендуется добавить серверную проверку
