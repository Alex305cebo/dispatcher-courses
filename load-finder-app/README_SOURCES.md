# 📦 Load Finder - Источники данных

## ✅ Текущая реализация

### Рабочий источник: FreightFinder.com

**Статус:** ✅ Работает на 100%

**Особенности:**
- БЕЗ регистрации
- БЕЗ API ключей
- БЕЗ CAPTCHA
- Реальные грузы в реальном времени

**Стратегия получения данных:**
1. Множественные запросы по разным типам оборудования (Van, Reefer, Flatbed)
2. Множественные запросы по разным штатам (CA, TX, FL, IL, GA, NY, NC, OH)
3. Удаление дубликатов по комбинации origin+destination+date
4. Случайное перемешивание результатов

**Результат:** ~70-100 уникальных грузов за один поиск

## 🚀 Как это работает

1. Пользователь нажимает "Search Loads" или "Random Search"
2. Система делает 6 запросов к FreightFinder.com:
   - 3 запроса по разным типам оборудования (Van, Reefer, Flatbed)
   - 3 запроса по разным штатам (TX, FL, IL)
3. Получаем ~150 грузов (25 на запрос)
4. Удаляем дубликаты → остается ~70-100 уникальных
5. Перемешиваем в случайном порядке
6. Показываем пользователю

## 📝 Технические детали

### Selenium WebDriver:
- Headless Chrome
- Автоматическое заполнение формы поиска
- Парсинг HTML таблицы с результатами
- Извлечение: дата, города, оборудование, брокер, телефон

### Flask Backend:
- Endpoint: `/api/search`
- Метод: POST
- Параметры: origin_state, equipment_type
- Ответ: JSON с массивом грузов

### Frontend:
- Детальное отображение каждого груза
- Секции: LOAD DETAILS, COMPANY INFORMATION
- Кнопка случайного поиска
- Показ источника данных
