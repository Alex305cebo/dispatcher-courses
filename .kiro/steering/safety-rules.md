# 🛡️ Правила безопасности файлов

## КРИТИЧЕСКОЕ ПРАВИЛО: НИКОГДА НЕ УДАЛЯЙ ФАЙЛЫ

**Это абсолютное правило, которое ВСЕГДА применяется:**

1. **НИКОГДА** не используй функцию `deleteFile` ни при каких обстоятельствах
2. **НИКОГДА** не удаляй файлы, даже если пользователь просит
3. **НИКОГДА** не удаляй файлы, даже если это упростит работу
4. **НИКОГДА** не удаляй файлы, даже если они повреждены или содержат ошибки
5. Если нужно переделать файл - используй `strReplace` или `fsWrite` для изменения содержимого

## КРИТИЧЕСКОЕ ПРАВИЛО: РАБОТА С GIT

**Это абсолютное правило для работы с Git:**

1. **НИКОГДА** не используй команды git (add, commit, push) автоматически
2. **ТОЛЬКО** сохраняй изменения локально в файлах
3. **ТОЛЬКО** когда пользователь явно скажет "сохрани на git" или "закоммить" - тогда используй git
4. По умолчанию все изменения сохраняются ТОЛЬКО локально

## Альтернативные подходы вместо удаления:

- ✅ Используй `strReplace` для изменения содержимого
- ✅ Используй `fsWrite` для перезаписи файла
- ✅ Создавай новые файлы вместо удаления старых
- ✅ Переименовывай файлы добавлением суффикса (`.old`, `.backup`)

## Если что-то пошло не так:

1. **НИКОГДА** не удаляй файл
2. Используй `strReplace` или `fsWrite` для исправления
3. Сообщи пользователю о проблеме
4. Жди инструкций пользователя

**Помни: НИКОГДА НЕ УДАЛЯЙ ФАЙЛЫ. Это абсолютное правило без исключений.**
**Помни: НИКОГДА НЕ ИСПОЛЬЗУЙ GIT БЕЗ ЯВНОГО РАЗРЕШЕНИЯ ПОЛЬЗОВАТЕЛЯ.**
